리액트 동작 원리에 대해서 깊이 있게 알고 싶어서 이 책을 읽게 됐다.
책을 다 읽은 소감은 리액트 뿐아니라 프론트엔드 개발자라면 배워야할 전반적인 내용에 대해서 설명해주는 지침서 같은 느낌이 들었다.
책에는 리액트의 여러 기능을 단순히 설명하는 것이 아니라 동작 원리와 내부적으로 어떻게 구현되어 있을지 예제 코드를 통해 자세하게 설명이 되어 있었다.
개인적으로 예제 코드 난이도가 조금 높았는데, 스터디 중간에 윤석님이 코드를 분석하는 것을 보면서 "아 저렇게 코드를 읽는거구나 ~~"하고 깨달음(?)을 조금 얻은 것이 기억난다.

책을 통해서 함수형 컴포넌트에서 사용하는 여러 훅의 원리에 대해서 알 수 있었고, 특히 내부적으로 자바스크립트 클로저를 이용해서 동작한다는 것을 알게됐다.
매번 리액트 코드를 작성할 때마다 `useEffect` 의존성 배열을 채우는 방법과 실행 시점에 대해 이해도가 부족했는데, 책을 읽으면서 `useEffect`가 실행되는 시점, 클래스 컴포넌트 mount 생명주기와 어떤 점이 다른지도 알 수 있었다.
또한, `useEffect` 의존성 배열이 변경된 것을 확인하는 데 사용되는 `shallowEqual` 함수의 구현도 직접 확인해 볼 수 있었다.

리액트 컴포넌트의 렌더링 과정에 대해서도 자세하게 설명되어 있었다.
리액트는 렌더, 커밋 2 단계를 거쳐서 컴포넌트를 렌더링하는데, 이때 리액트 파이버가 사용된다는 것을 알 수 있었다.(이 부분은 개념이 너무 어려워서 완벽히 이해하진 못했다.)

브라우저 렌더링, 개발자 도구를 이용한 성능 측정, 보안에 대한 내용, CI/CD에 대한 내용까지 정말 다양한 내용을 다루고 있었는데, 한번에 이해하긴 어려웠고 필요할 때 마다 꺼내서 자주 읽어봐야겠다.
무엇보다 리액트에 대해서 공부했는데, 실무에 사용하고 있는 vue 렌더링에 대해서도 다시 한 번 생각해보게 되었다.
프레임워크를 단순히 사용하는 것이 아니라 동작 원리에 대해 이해하고 사용하면 더 좋은 코드를 작성할 수 있다는 것을 알게됐다.
